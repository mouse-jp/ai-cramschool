# 公開（デプロイ）手順 — Streamlit Community Cloud（無料）

アプリをインターネットに公開し、`https://〇〇〇.streamlit.app` のような**公開URL**で
誰でもアクセスできるようにする手順です。無料で始められます。

---

## ⚠️ 公開前の最重要チェック：費用と不正利用

このアプリは Gemini API（有料・従量課金）を使います。**公開して誰でも新規登録できる状態**にすると、
登録した人全員が**あなたのGeminキーでAIを使える**＝**あなたに課金される**ことになります。
悪意ある利用や想定以上のアクセスで、**料金が膨らむリスク**があります。

公開前に、次のどれかで必ず対策してください：

1. **ユーザーが自分のGeminキーを入力する方式**（運営の費用ゼロ）
   → クラウドのSecretsに `GEMINI_API_KEY` を**入れない**。各ユーザーがサイドバーに自分のキーを入力します。
   （アプリには既にサイドバーのキー入力欄があります）

2. **運営が負担するが、上限と人数を絞る**
   → Google Cloud（またはAI Studio）の請求で**予算上限・アラート**を設定。
   → Supabase の Authentication で **新規登録を一旦オフ（招待制）**にし、信頼できる人だけ登録。

3. **まだ公開しない**（ローカル／限定メンバーのみで運用）

> このアプリは受験生（未成年を含む）向けです。公開時は**利用規約・プライバシーポリシー**の用意、
> および**過去問データの著作権**（公開配布してよいか）の確認もおすすめします。

### 費用上限の設定（運営負担で公開する場合）— 推奨の安全策

1. **月額の上限（Spend Cap）を設定する**
   - **Google AI Studio** にログイン → 左の **「Spend」** ページ → **「Monthly spend cap」→ Edit spend cap** で
     **月額の上限ドル**を設定（例：まずは数ドル/月など小さく）。上限に達するとそれ以上課金されません
     （※反映に約10分のラグがあり、その間はわずかに超過する可能性あり）。
   - これにより「想定外の高額請求」を防げます。
2. **最初は招待制にする（人数を絞る）**
   - Supabase → Authentication → **「Allow new users to sign up」を一旦 OFF**。
   - あなたが信頼できる人だけ、こちらでアカウントを作って渡す（または一時的にONにして登録してもらう）。
3. **使いすぎ対策（将来）**
   - 営利化に向けては、**ユーザーごとの1日あたりAI利用回数の上限**や、**有料プラン（Stripe等）での課金**を
     段階的に追加していくのがおすすめ。これは別途実装で対応できます。

## 手順

### ステップ1：GitHubにコードを上げる
1. GitHub（https://github.com）でアカウント作成 → 新しいリポジトリを作成（Privateで可）。
2. プロジェクトフォルダを push する。
   - **`secrets.toml` は絶対に上げない**（`.gitignore` で `.streamlit/` を除外済みなのでOK）。
   - 大きめのデータJSON（`past_exams_db.json` 等）も一緒に上げます（アプリが読むため）。

### ステップ2：Streamlit Community Cloud に接続
1. https://share.streamlit.io を開き、**GitHubでサインイン**。
2. **「New app」** → 先ほどのリポジトリ／ブランチ／**Main file path に `app.py`** を指定。

### ステップ3：Secrets（鍵）を設定
「Advanced settings → Secrets」に、ローカルの `secrets.toml` と同じ内容を貼ります
（**費用方式によって `GEMINI_API_KEY` を入れるか決める**＝上の対策1なら入れない）：

```toml
SUPABASE_URL = "https://xxxx.supabase.co"
SUPABASE_KEY = "eyJ...（anon public）"
# GEMINI_API_KEY = "..."   # ← 運営負担にする場合だけ入れる
```

### ステップ4：デプロイ
**「Deploy」** を押すと数分でビルドされ、`https://〇〇〇.streamlit.app` が発行されます。これが公開URLです。

### ステップ5：Supabase 側の仕上げ
1. テスト中にオフにした **「Confirm email」を ON に戻す**（公開時はメール確認推奨）。
2. （Googleログイン等を使う場合）Authentication → URL Configuration の
   **Redirect URLs / Site URL** に、発行された `https://〇〇〇.streamlit.app` を追加。

---

## 公開後の注意

- **管理ツール `db_manager.py` は公開アプリに含めない**（過去問の編集など管理機能。手元で使う）。
- **共有ファイルに書き込む生成機能**（頻度つよつよ単語の意味生成など、`base_lexicon.json` を書き換える系）は、
  クラウドでは保存が消えます（サーバーのファイルは一時的なため）。
  → これらは**手元で実行してから GitHub にコミット**して反映する運用にします。
- ユーザーの学習データ（単語帳など）は **Supabase に保存**されるので、再起動・別端末でも残ります。

## まとめ

- 公開URL＝`https://〇〇〇.streamlit.app`（Streamlit Community Cloudが発行）
- アプリ本体は GitHub のコードから自動デプロイ。コードを更新して push すると自動で反映。
- **費用方式（対策1〜3）を決めてから公開**するのが最重要です。
